# Plan TDD — `z web` dashboard (PRD-web)

Date: 2026-04-25
PRD source: `docs/PRD-web.md`

## Approche

Tracer bullets verticales (skill TDD). Un test → une impl → boucle. **Pas** d'horizontal slicing.

## 7 tracer bullets (ordre strict)

1. **z-core — `dashboard_groups`** : `(sessions, projects, activity, counts) -> Vec<ProjectGroup>`. Filtre z-managed (project match) + non-remote (project.host is None). Groupes tri alpha par project name ; sessions dans groupe tri last-attach desc. Pur.
2. **z-core — `zellij_session_url(host, port, session)` -> String**. Encoding slashes/colons/non-ASCII, IPv6 hosts. Pur.
3. **z-web — template index (maud)**. Fixture `Vec<ProjectGroup>` → substring assertions (nom projet, branch, badge, lien).
4. **z-web — empty state**. Zero sessions → message hint + liste projets connus.
5. **z-web — HTTP smoke** via `axum::Router` + `tower::ServiceExt::oneshot`. Mock `SessionManager`/`ProjectStore`. `GET /` → 200, `text/html`, contient les noms attendus.
6. **z-cli — `z web status`** avec mock `WebDaemon`. 3 cas : all up / all down / partial → stdout + exit code 0/1/2.
7. **z-cli — `z web rotate`** avec mock. Ordre `--revoke-token` → `--create-token`, fichier state réécrit.

Après chaque bullet : stop, je fais valider par l'utilisateur avant de passer au suivant.

    pub name: String,            // "myapp:main"

    pub branch: String,
    pub last_attach: Option<u64>,

    pub sessions: Vec<DashboardSession>,

    activity: &HashMap<String, u64>,

pub fn zellij_session_url(host: &str, port: u16, session: &str) -> String;
